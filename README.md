# VscodeShellLayout

SengenUIの上に構築されたVSCode風ドッキングUI外殻ライブラリ。メニューバー・アクティビティバー・エディタエリア（タブDnD・ペイン分割）・パネル・右サイドバー・ステータスバーを提供する。

元はPokemonBattleAIリポジトリ（`megadenryu_support_ui/submodules/VscodeShellLayout`）内で開発されていたものを、複数プロジェクト（PokemonBattleAI / AgentRoom / MonogatariAI）で共用するため独立リポジトリ化した。それ以前の変更履歴はPokemonBattleAIリポジトリにある。

## 利用方法

npmパッケージとしては配布していない。ホストリポジトリにgit submoduleとして取り込み、TypeScriptソースを直接参照する。

前提（3点セット）:
1. 本リポジトリを `submodules/VscodeShellLayout` に配置
2. SengenUI（git@github.com:megaDenryu/SengenUI.git）を隣の `submodules/SengenUI` に配置（`package.json` が `file:../SengenUI` で参照するため、並び位置が契約）
3. ホスト側のViteに `@vanilla-extract/vite-plugin` を導入し、`resolve.alias` で `sengen-ui` を `submodules/SengenUI` に向ける

エントリポイントは `src/index.ts`。主要APIは `外殻レイアウト`（全スロットをコンストラクタ注入するLV2 Orchestrator）。

## 構成とデザインの自由化

ライブラリは機能（タブDnD・ペイン分割・トグル操作）を提供し、構成とデザインはアプリ側が決める、という方針。

### 領域のオプション化

`外殻レイアウトオプション` の `メニューバー表示?` / `ステータスバー表示?`（いずれも省略時 `true`）を `false` にすると、
該当領域のDOM自体が生成されない（data-attributeでの非表示ではなくゼロ高さ）。

```typescript
new 外殻レイアウト({
    タイトル: "アプリ名",
    アクティビティ項目一覧: [...],
    メニューバー表示: false,   // 独自ヘッダーに置き換える場合
    ステータスバー表示: false,
});
```

### トグルボタンの独立コンポーネント化

右サイドバー/パネルの開閉は、メニューバー内蔵ボタンだけでなく `サイドバートグルボタン` / `パネルトグルボタン`
（いずれもLV1拡張、`index.ts` からexport）を任意のスロット（ステータスバー・アクティビティバー下部等）に配置して操作できる。

配線先は `Iレイアウトトグル操作`（`サイドバーを切り替える()` / `パネルを切り替える()`）で、
`外殻レイアウト` 自身がこのインターフェースを実装するため、シェルのインスタンスをそのまま渡せる。

```typescript
const サイドバートグル = new サイドバートグルボタン();
const シェル = new 外殻レイアウト({
    タイトル: "アプリ名",
    アクティビティ項目一覧: [...],
    ステータスバー左: サイドバートグル,       // 構築時オプションで好きなスロットに配置
    メニューバートグルボタン表示: false,      // メニューバー内蔵の同機能ボタンは消す
});
サイドバートグル.配線する(シェル);            // 構築後に1回だけ配線する
```

### アクティビティバー下部の注入化

固定の設定ボタンは廃止し、`アクティビティバー下部項目一覧?: アクティビティ項目[]` で任意のボタン列に置き換えられる。
省略時は旧挙動（固定の設定ボタン + `on設定クリック`）のまま動作する（非推奨）。

```typescript
new 外殻レイアウト({
    タイトル: "アプリ名",
    アクティビティ項目一覧: [...],
    アクティビティバー下部項目一覧: [
        { id: アクティビティID("設定"), ラベル: "設定", アイコン: 設定アイコン },
    ],
});
// クリックは onアクティビティ選択(id) で受け取る（on設定クリック は非推奨）
```

### テーマのCSSカスタムプロパティ化

`テーマ配色` の全トークンは各コンポーネントのstyle.css.tsで `var(--vsl-xxx, デフォルト値)` として参照される。
`外殻レイアウトオプション.テーマ?: Partial<テーマ配色>` を渡すと、指定したトークンだけがルート要素に
CSSカスタムプロパティとして適用される（`setCssVariable` 経由。未指定トークンはCSS側のフォールバックのまま）。
テーマを一切指定しない場合は現行デフォルト（ネイビー×イエロー）と完全に同一の見た目になる。

```typescript
new 外殻レイアウト({
    タイトル: "アプリ名",
    アクティビティ項目一覧: [...],
    テーマ: { アプリ背景: "#101820", ブルー: "#4da3ff" },
});
```

### マトリョーシカシェル

エディタエリアのタブに、別アプリの `外殻レイアウト` を1タブとしてホストする構成の名前。
CSSカスタムプロパティはDOMカスケードするため、内側シェルのルートに外側と異なるテーマを渡すだけで、
内外の配色を描き分けられる（内側の変数上書きは内側のsubtreeにしか波及しない）。

**規範: マトリョーシカシェルを組むときは、内側シェルには必ず別テーマを当てて階層を描き分けること。**
同一テーマのまま入れ子にすると、どこからどこまでが内側アプリなのかユーザーが視覚的に判別できない。

```typescript
外側シェル.タブを追加する("内側アプリ", "内側アプリ", new 外殻レイアウト({
    タイトル: "内側アプリ",
    アクティビティ項目一覧: [...],
    テーマ: { アプリ背景: "#1e1e2e", ネイビー: "#2a2a3d" },  // 外側と異なるテーマを必ず指定する
}));
```
